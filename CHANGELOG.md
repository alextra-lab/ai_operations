## [0.1.0] - 2026-04-28

### Added

- Initial public release of the AI Operations Platform.
- MIT [`LICENSE`](LICENSE), [`SECURITY.md`](SECURITY.md),
  [`CONTRIBUTING.md`](CONTRIBUTING.md), and
  [`CODE_OF_CONDUCT.md`](CODE_OF_CONDUCT.md).
- GitHub repo metadata: issue templates, PR template, Dependabot config,
  CI and license-scan workflows.
- License classifier and project URLs in [`pyproject.toml`](pyproject.toml).
- README quick-start now documents the external `observability` Docker
  network and the default `linux/arm64` compose platform.

### Changed

- Replaced PyMuPDF (AGPL-3.0 / Artifex commercial) with the existing
  permissive PDF stack (`pdfplumber`, `PyPDF2`, `pytesseract`) — the package
  was previously declared in [`src/corpus_svc/requirements.txt`](src/corpus_svc/requirements.txt)
  but never imported in source.
- Migrated CI from GitLab (`.gitlab.ci.yml`) to GitHub Actions
  (`.github/workflows/`).

### Removed

- `export_env_template()` generator from [`src/shared/config/loader.py`](src/shared/config/loader.py)
  and the underlying `ConfigManager.export_env_file()` /
  `BaseConfig.to_env_dict()` methods. The generator dumped live environment
  values into a file labelled `template`, which conflicted with the
  hand-maintained source of truth at
  [`config/env/env.template`](config/env/env.template) (per ADR-071) and
  produced unprefixed variable names that no service consumed. Removing it
  also removes a class of "live secret accidentally written to disk" bug.
- Stale `.gitlab.ci.yml` (license-scan job is now in
  `.github/workflows/license-scan.yml`).

### Notes for downstream consumers

- `license-report.csv` should be regenerated by running
  `pip-licenses --format=csv --with-urls --with-description > license-report.csv`
  inside an environment with the runtime dependencies installed. The
  shipped report has been pruned manually to remove the PyMuPDF row.
- The CI workflow regenerates the license report on every push.
